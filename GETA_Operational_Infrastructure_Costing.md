# GETA Operational Infrastructure Costing

**What GETA needs to host resources, run the enquiry line, and sustain the service**

**Context:** GETA is a parent-led national organisation. The current GO8264 budget sits with UNSW as lead. If GETA is to hold and serve the resources from their own web presence and operate the enquiry/call line, the following infrastructure and staffing is needed. This is the "run it in the real world" layer.

## Key Decision Points

### 1. Where does this money come from?

The current GO8264 budget doesn't fund GETA operations. Options:

| Option | Pros | Cons |
|---|---|---|
| **Include in grant budget as a sub-activity** | Clean, auditable, directly funded | Increases total ask; UNSW is lead so funds flow through them |
| **GETA co-contribution (in-kind or cash)** | Strengthens bid (demonstrates commitment) | GETA is a parent-led org - cash co-contribution may not be feasible |
| **Hybrid: grant funds setup + Year 1-2, GETA absorbs Years 3-5** | Matches Category B sustainability narrative | GETA needs revenue/funding to sustain |
| **GETA applies for complementary funding** (e.g. ILC, philanthropic) | Diversifies funding base | Timing risk; not guaranteed |

### 2. Does GETA host the resources, or does UNSW host and GETA links?

| Model | GETA Hosts | UNSW Hosts, GETA Links |
|---|---|---|
| Cost to GETA | Higher (full web infrastructure) | Lower (just maintain own site + links) |
| Control | GETA controls content presentation | UNSW controls; GETA is a referral point |
| Sustainability | GETA can maintain post-grant | Dependent on UNSW continuing to host |
| Community trust | Families may trust a parent-led org more | UNSW has institutional credibility |
| **Recommended** | **For the resource library front-end** | **For the backend/KM platform** |

The strongest model: UNSW builds and maintains the backend (KM platform, AI infrastructure, data). GETA hosts a consumer-facing front end that pulls from the shared backend. Families interact with GETA's trusted brand. UNSW provides the institutional backbone. Both can survive independently if needed.

### 3. Who staffs the call line?

| Role | Who | Cost Impact |
|---|---|---|
| **Triage & first response** | AI (Area 2) + GETA webpage manager | Already in AI budget + webpage manager cost above |
| **Peer connector (lived experience)** | GETA parent volunteers with stipend/honorarium | $5,000-10,000/yr for stipends + training |
| **Specialist information** | Senior Info & Advice Lead (UNSW, Activity 2) | Already in grant budget |
| **Safeguarding escalation** | Lived Experience Lead (UNSW, Activity 1) | Already in grant budget |
| **Clinical queries** | Consortium clinical partners (SCHN, RCH, CHQ) | Already in consortium arrangements |

The AI triage layer (Area 2) is what makes this viable for GETA. Without it, running a national call line from a parent-led org with no dedicated call staff is not sustainable. With AI handling classification, drafting replies, and routing - GETA's webpage manager + trained parent volunteers can handle the human touchpoints.
